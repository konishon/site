Reading the Landscape with AI

A lightweight experiment combining image captioning, NLP, and remote sensing to produce human-readable change summaries.

Nishon TandukarGeoAI research and data specialist
In this article The ideaAcademic contextVision-language modelsThe workflowWhy this is usefulLimitationsWhat comes nextReferences

Remote sensing change detection is usually treated as a pixel problem. We compare two images from different dates, identify changed areas, and produce a binary mask or classified map.

This is useful when the goal is measurement: how many pixels changed, where they changed, and how much area was affected. Change detection is commonly framed as assigning change or no-change labels to pixels in co-registered images from different times—especially for urban expansion, land-use monitoring, deforestation, and damage assessment.

But many real users also want something more direct: what actually changed on the ground?

Did open land become a built-up area? Did vegetation disappear? Did new roads or parking lots appear? Did a construction site turn into a commercial complex?

This experiment explores a lightweight semantic change detection workflow. Instead of starting with pixel differencing, I use a vision-language model to describe two aerial images in natural language. I then apply NLP techniques to compare the generated descriptions and produce a readable summary of land-cover and land-use change.

Important distinctionThis is not a replacement for classical change detection models. It is a semantic explanation layer—a way to turn visual differences into readable observations.

The idea

The key shift is that the image pair is converted into text before comparison. Once the visual content is represented as language, we can compare meaning, extract land-use features, and generate a readable summary.

Image at Time 1 → Vision-Language Model → Caption 1 Image at Time 2 → Vision-Language Model → Caption 2 Caption 1 + Caption 2 → NLP comparison → Change summary

The pipeline has three main parts:

  1. Image captioning using a vision-language model
  2. NLP-based feature extraction from generated captions
  3. Semantic comparison for change detection

Academic context

This experiment sits between two research areas. The first is remote sensing image change detection. Modern methods learn from bitemporal image pairs and produce pixel-level change maps. The field has moved from hand-crafted features toward CNNs, attention mechanisms, transformers, and foundation-model-inspired pipelines.

The Bitemporal Image Transformer (BIT) is a useful reference because it argues that high-resolution change detection needs both semantic understanding and spatial-temporal context—not just raw image differencing.

The second area is vision-language representation learning. CLIP showed that large-scale image-text supervision can produce visual models that transfer across tasks using natural-language prompts. Recent remote sensing work has begun exploring similar language-guided approaches to semantic and open-vocabulary change detection.

My method is much lighter than those systems. It does not train a change detection model. It asks a practical question: can captioning plus NLP provide a readable first-pass interpretation of change?

What is a vision-language model?

A vision-language model (VLM) connects visual information with language. It can take an image as input and answer questions, generate captions, classify objects, or describe a scene.

In a geospatial context, a VLM can inspect an aerial image and describe visible features such as buildings, roads, vegetation, water bodies, agricultural fields, construction sites, parking lots, and industrial areas. Instead of raw pixel values, it produces a description humans can read.

For this experiment, I used the vikhyatk/moondream2 model from Hugging Face:

from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained(
    "vikhyatk/moondream2",
    revision="2025-06-21",
    trust_remote_code=True,
    device_map={"": "cuda"},
)

The workflow

Step 1: Load a pair of aerial images

The experiment uses a pair of aerial images from the LEVIR-CD change detection dataset. Each pair contains images of the same area captured at different times.

from PIL import Image
import matplotlib.pyplot as plt

pair_number = 5
image_path1 = f"/kaggle/input/levir-cd-change-detection/LEVIR-CD+/train/A/train_{pair_number}.png"
image_path2 = f"/kaggle/input/levir-cd-change-detection/LEVIR-CD+/train/B/train_{pair_number}.png"

image1 = Image.open(image_path1)
image2 = Image.open(image_path2)

The first image contains mostly open land, roads, sparse vegetation, and a development area. The second shows a much more developed scene, with large buildings, parking lots, and organized road infrastructure.

Step 2: Generate captions

caption1 = model.caption(image1, length="long")["caption"]
caption2 = model.caption(image2, length="long")["caption"]

This is where the workflow becomes language-based. Instead of comparing image arrays directly, we now have two text descriptions. The captions already offer a quick qualitative reading—but they still need to be compared.

Step 3: Bring in NLP

NLP turns free-form captions into structured signals. The goal is to identify features shared by both images, features unique to each date, and how similar the descriptions are semantically.

from sentence_transformers import SentenceTransformer, util

embedding_model = SentenceTransformer("all-MiniLM-L6-v2")
embedding1 = embedding_model.encode(caption1, convert_to_tensor=True)
embedding2 = embedding_model.encode(caption2, convert_to_tensor=True)

cosine_score = util.cos_sim(embedding1, embedding2).item()
semantic_difference = 1.0 - cosine_score

A high similarity score suggests that the captions describe comparable scenes. A lower score suggests meaningfully different scenes. But semantic similarity is not physical change: two captions can use the same words while hiding major changes in amount, position, or density.

Extract land-use and land-cover features

I used compact keyword lists and spaCy to extract relevant nouns, adjectives, and verbs from each caption.

def extract_land_components(text):
    doc = nlp(text)
    components = {
        "land_cover": set(),
        "land_use": set(),
        "descriptors": {t.lemma_.lower() for t in doc if t.pos_ == "ADJ"},
        "actions": {t.lemma_.lower() for t in doc if t.pos_ == "VERB"},
    }
    for token in doc:
        if token.pos_ in {"NOUN", "PROPN"}:
            lemma = token.lemma_.lower()
            if lemma in LAND_COVER_KEYWORDS:
                components["land_cover"].add(lemma)
            if lemma in LAND_USE_KEYWORDS:
                components["land_use"].add(lemma)
    return components

The most useful comparison is not the raw set of words, but the difference between them:

Shared features: • roads • vegetation Features reduced or lost: • open fields • sparse land cover Features gained: • buildings • parking lots • commercial or industrial development

Step 4: Calculate a feature-change score

To estimate how different the extracted feature sets are, I used Jaccard distance: one minus the overlap divided by all unique features.

features_1 = features_a["land_cover"] | features_a["land_use"]
features_2 = features_b["land_cover"] | features_b["land_use"]

intersection = len(features_1 & features_2)
union = len(features_1 | features_2)

feature_change_score = None if union == 0 else 1.0 - (intersection / union)
Implementation noteIf both captions yield zero matched keywords, that is insufficient evidence—not “no change.” Otherwise, extraction failure can be mistaken for landscape stability.

Step 5: Produce a readable summary

The first image appears to show mostly open land, sparse vegetation, roads, and limited development. The second image appears to show a more developed commercial or industrial area, with large buildings, parking lots, and organized road infrastructure. Overall, the area appears to have changed from a more open or rural landscape into a more built-up developed landscape.

Why this is useful

Classical change detection remains essential for measurement, localization, and validation. But its outputs often require technical interpretation. A language layer can complement these methods with:

  • Quick scene summaries
  • Land-use and land-cover explanations
  • Human-readable reports
  • Searchable text descriptions
  • First-pass interpretation for analysts

Instead of ending with a binary mask, the system could say: “Open land and sparse vegetation were replaced by large commercial buildings, parking areas, and internal roads.” That is much easier to include in reports, dashboards, and decision-support tools.

Where this approach falls short

1. Captions can miss important details

Vision-language models may miss small objects, confuse land-use categories, or describe an object too generally.

2. Captions can hallucinate

A model may mention plausible features that are not actually present—especially where aerial object scale and resolution are ambiguous.

3. Text similarity is not spatial change

Captions may remain semantically similar even when the spatial layout has changed significantly.

4. Keyword lists are limited

A transparent keyword approach is also brittle. It can miss multi-word concepts such as “parking lot,” “construction site,” or “power plant.”

5. There is no pixel-level localization

This workflow can suggest what changed, but not precisely where. Operational use should combine it with segmentation, object detection, or classical change detection.

A better next version

The next version should ask the VLM to compare the image pair directly, rather than captioning each image independently:

Compare these two aerial images of the same location. Describe only visible land-use and land-cover changes. Return JSON with: • unchanged_features • lost_features • gained_features • likely_change_type • confidence

Another improvement is to validate the semantic output against ground-truth change masks from datasets like LEVIR-CD. That would answer two important questions: did the model describe the correct kind of change, and did that description correspond to actual changed pixels?

A practical hybrid workflow

Satellite or aerial image pair ↓ Classical change detection or segmentation model ↓ Changed regions ↓ Vision-language model ↓ Semantic explanation ↓ Human-readable change report

Here, the classical model finds where change happened, while the VLM and NLP layer explain what the change means. That combination is much stronger than captions alone.

Conclusion

This experiment demonstrates a simple but useful idea: aerial-image change detection does not always have to end with a mask. Sometimes, we also need language.

By using a vision-language model, we can convert aerial imagery into scene descriptions. With NLP, we can compare those descriptions and extract semantic signals about land-cover and land-use change.

Pixel-level change tells us where the image changed. Semantic change helps us explain what changed.

For geospatial applications, both views matter.
